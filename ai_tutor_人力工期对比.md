# AI Tutor 工期对比：MVP 6–10 人团队 vs 精简 3–5 人团队

本文档用于说明在不同人力规模下，AI Tutor MVP 的工期差异与关键影响因素，供内部评审与资源决策使用。

---

## 1. 适用范围与假设

- **范围**：离线复盘 MVP + 基础报告仪表盘（不含实时 Tutor）
- **数据规模**：50–200 节课样例规模
- **合规**：授权与隐私流程已准备（不作为关键路径）
- **团队角色**：后端/数据/算法/前端/产品各覆盖

---

## 2. 工期对比（核心结论）

| 模块 | 6–10 人团队 | 3–5 人团队 | 关键差异原因 |
|---|---:|---:|---|
| 数据接入与清洗 | 2–3 周 | 4–6 周 | 并行ETL与质检能力 |
| 对齐与ASR | 4–6 周 | 8–12 周 | 模块并行开发受限 |
| 特征层与模型基线 | 4–6 周 | 8–12 周 | 训练/评估周期拉长 |
| 报告与仪表盘 | 4–6 周 | 8–10 周 | 前后端交付节奏 |
| 集成与验收 | 2–3 周 | 4–6 周 | 集成测试资源不足 |
| **总工期** | **约 4–6 个月** | **约 7–10 个月** | 关键链条难并行 |

---

## 3. MVP 交付节奏示例

### 6–10 人团队（推荐）
1. **第 1 月**：ETL + 对齐基础 + 初版 ASR
2. **第 2 月**：话轮识别 + 特征层 + 报告雏形
3. **第 3 月**：KT 基线 + 仪表盘 + 试点验证
4. **第 4–5 月**：优化迭代 + 初步效果评估

### 3–5 人团队（精简）
1. **第 1–2 月**：ETL + 对齐 + ASR
2. **第 3–4 月**：特征层 + 报告生成
3. **第 5–6 月**：仪表盘 + 试点验证
4. **第 7–9 月**：模型优化 + 评估迭代

---

## 4. 关键影响因素

- **并行度**：团队人数决定模块并行开发能力
- **数据准备耗时**：清洗与对齐是最长链路
- **算法训练迭代**：小团队训练/调参周期更长
- **测试与验收**：资源不足会拉长试点验证时间

---

## 5. 结论与建议

- **6–10 人团队**可在 4–6 个月完成 MVP 闭环，适合对结果有明确时间要求的场景。
- **3–5 人团队**适合探索期或资源受限场景，但整体工期可能拉长至 7–10 个月。
- 若目标包含实时 Tutor，建议在 MVP 稳定后另行评估并追加人力与工期。
